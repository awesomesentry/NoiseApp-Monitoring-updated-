 System Architecture
High-Level Architecture (3 Layers)
Layer 1 – Edge / Classroom Layer
(Installed in each classroom)
Hardware Components:
ESP32 microcontroller
INMP441 digital microphone (I2S – real audio sensing)
Traffic light LEDs (Green / Yellow / Red)
Buzzer
Status LED (Wi-Fi & system state)
Internal flash storage (NVS)
Responsibilities:
Continuous noise sampling (100–200 ms)
dB computation and smoothing
Escalation-based alert logic
Event-triggered audio capture
Offline event buffering
Layer 2 – Backend / Server Layer
(Centralized and secure)
Components:
REST API server
PostgreSQL / Supabase database
Secure audio storage
Authentication & RBAC module
Responsibilities:
Store noise events and metadata
Manage audio retention
Enforce access control
Provide data to the web dashboard
Layer 3 – Web Application Layer
(Teachers and Administrators)
Components:
Web dashboard (React / Vue / HTML)
Charts and analytics
Role-based UI views
Responsibilities:
Data visualization
Event validation
Analytics and reporting
System configuration (admin only)
 Complete System Flow
Step 1: Continuous Noise Sampling
INMP441 samples audio every 100–200 ms
Converts signal to dB values
Applies moving-average smoothing
Step 2: Threshold Evaluation
Example Thresholds:
Green: < 60 dB
Yellow: 60–74 dB
Red: ≥ 75 dB
IF dB < Green → Green LED
ELSE IF dB < Red → Yellow LED
ELSE → Red LED

​
Step 3: RED-Level Trigger (Event-Based)
IF dB ≥ Red threshold
FOR ≥ 2–3 seconds
THEN confirm RED event

​
 Prevents false positives
Step 4: Escalation-Based Buzzer Logic
Loud ≥ 2 seconds → 1 short beep
Loud ≥ 30 seconds → 2 short beeps
Loud ≥ 60 seconds → Major violation logged
 No continuous buzzing
 Classroom-friendly alerts
Limits:
Max beeps per event
Max total alert duration (e.g., 30 seconds)
Step 5: Audio Capture
Triggered only during confirmed RED events
Records 3–5 seconds only
No live listening
No continuous recording
Step 6: Offline-Safe Data Logging
When Wi-Fi is unavailable:
Noise Event → Stored in ESP32 Flash (NVS)
Wi-Fi Restored → Automatic Cloud Sync

​
 No data loss
 No reboot required
Step 7: Web App Visualization
Users log in
Data displayed based on role permissions

ESP32 pins to wire DS3231
Default ESP32 I2C pins (what I set in code):
SDA = GPIO 21
SCL = GPIO 22
VCC = 3.3V (most DS3231 boards accept 3.3V or 5V, but use 3.3V on ESP32)
GND = GND
I moved DS3231 I2C off the status LED pins:
DS3231 SDA = GPIO32
DS3231 SCL = GPIO4
 IMPORTANT HARDWARE NOTES (PLEASE READ)
 Do NOT power INMP441 with 5V
 Do NOT share 5V MP3 power with ESP32 GPIO
 Common GND is mandatory
 MP3 boot delay (8s) in code is correct
 SD SPI speed reduced → good for stability
